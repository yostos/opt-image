# opt-image 想定ユースケース

最終更新: 2026-06-18

ローカル画像を **減色・リサイズ・署名して最適化フォーマットで出力**する CLI ツール。
旧 `~/bin/opt-image` の Optimage 依存を排し、無料・arm64 ネイティブの CLI ツール
（`magick` / `cjpeg`(mozjpeg) / `cwebp` / `oxipng`）だけで組む。

関連: 作り直しの経緯・背景は [`HANDOFF.md`](./HANDOFF.md) を参照。

## このツールの根幹: 可逆か非可逆か

`opt-image` の使い分けは「画像をどの圧縮方式に載せるか」で決まる。
判断軸は**フォーマット名（jpg/webp/png）ではなく、圧縮が非可逆か可逆か**である。

- **非可逆（JPEG）**: 写真・グラデーションのように色が連続的に変化する画像に向く。
  知覚品質ベースで強く圧縮できるが、エッジや文字・平坦色は**圧縮アーティファクト
  （モスキートノイズ・にじみ）**が出る。世代劣化もある。
- **可逆（WebP / PNG）**: イラスト・スクリーンショット・ロゴのように
  **輪郭がはっきり・色数が限られる**画像に向く。エッジや文字が崩れない。
  色数を絞る（減色）ことでサイズを大きく削減できる。

> 写真を PNG にしても無駄に大きいだけ、線画を JPEG にすると輪郭が汚れる。
> **対象画像の性質で可逆/非可逆を選ぶ**のが最初の判断。

| | 非可逆 | 可逆 |
|---|---|---|
| フォーマット | JPEG | WebP / PNG |
| 対象画像 | 写真・グラデーション | イラスト・スクショ・ロゴ・文字 |
| 強み | 連続階調を高圧縮 | エッジ・平坦色を無劣化 |
| 弱み | エッジ/文字がにじむ・世代劣化 | 連続階調はサイズが膨らむ |
| エンコーダ | `cjpeg`（mozjpeg） | `cwebp` / `oxipng` |
| 減色 `-c` | 使わない | 使う（任意） |

## UC1: 写真を非可逆 JPEG で最適化

写真・グラデーション主体の画像を、知覚品質を重視した `mozjpeg`（`cjpeg`）で
軽量化する。旧ツールの主用途（Optimage に投げていた処理）の置き換え。

> 当初は `cjpegli`（jpegli）を想定したが、homebrew で入手できなかったため
> 同じく知覚最適化された定番エンコーダ `mozjpeg` を採用した。経緯は
> [`HANDOFF.md`](./HANDOFF.md) 参照。

**経路**: `magick`（リサイズ＋署名、中間 PPM）→ `cjpeg`（mozjpeg）

```bash
# 既定。1800px に収めて JPEG 最適化
opt-image -f jpg photo.png

# 公開用に暗色署名を入れる（明るい背景向け）
opt-image -f jpg -s dark photo.jpg

# 解像度を保ったまま最適化だけ
opt-image -f jpg -n large.jpg
```

- `-c`（色数）は JPEG では意味を持たない。指定してもエラー扱いとする想定。

---

## UC2: 線画・スクショを可逆 WebP / PNG で最適化

イラスト・スクリーンショット・ロゴなど**色数が限られる画像**を、エッジを崩さず
軽量化する。必要なら**減色（`-c`）**でさらに縮める。

**経路**: `magick`（リサイズ＋署名、`-c` 指定時は `-colors N` も）→ `cwebp` / `oxipng`

```bash
# 配信用に WebP（減色なし＝フルカラー可逆）
opt-image -f webp logo.png

# 256 色に減色して WebP
opt-image -f webp -c 256 illust.png

# 互換性重視で PNG、4096 色で階調を保持
opt-image -f png -c 4096 screenshot.png

# 白黒線画を 2 色まで落として最小化
opt-image -f png -c 2 lineart.png
```

### WebP と PNG の選び分け

両者とも可逆側。出力コンテナの違いだけ。

- **WebP**: 同条件で PNG より小さい。Web 配信向け。
- **PNG**: 互換性が最も広い。古い環境・素材受け渡し向け。

### `-c`（色数）未指定の挙動

`-c` を省略した場合は**減色しない（フルカラーのまま可逆圧縮）**。
色数を意図的に絞りたいときだけ `-c` を付ける。

## パラメータ（UC 横断）

ユースケースを決めるのは可逆/非可逆だけ。以下は両 UC に共通のつまみ。

### `-c <色数>` 減色（UC2 のみ有効）

| 値 | 用途の目安 |
|---|---|
| `2` | 白黒線画・2 値のロゴ |
| `16` | 単純なアイコン・少色イラスト |
| `125` | 5³。中間色を含むイラスト |
| `256` | パレット上限。一般的なスクショ・イラスト |
| `4096` | 16³。階調を多く残したいイラスト |

- 許可リストは `2 / 16 / 125 / 256 / 4096`。**範囲外はエラー**。
- 主軸は `magick -colors N`（256 超も統一的に扱える）。
  `≤256` のときだけ高品質な `pngquant` に切り替える余地は残してよい。
- `pngquant` はパレット PNG 前提で**上限 256 色**のため 4096 色は不可。

### `-p <pixels>` / `-n` リサイズ

- `-p <pixels>`: 長辺をその画素数に**収める（縮小のみ。指定より小さい画像は拡大しない）**。既定 `1800`。
- `-n`: リサイズしない（元解像度を保持）。
- 両者は排他。`-n` が優先。

### `-s <white|dark>` 署名

画像右下に署名とライセンス表記を入れる。

- `-s white`: 白文字（暗い背景向け）。
- `-s dark`: 暗色文字（明るい背景向け）。
- 署名文字列: `© 2026 Toshiyuki Yoshida` / `CC BY-NC-SA 4.0`

> 旧ツールの `-s` / `-sd` は getopts の解釈に依存した壊れやすい作りだった。
> 新版は `-s` に色引数を取る形に統一する。

### `-h` ヘルプ

使い方を表示して終了。

## 入出力の仕様

- **元ファイルを上書き**する（旧ツール踏襲）。処理前に `<input>.bak` を作成。
- フォーマット変更で**拡張子も変わる**（例: `photo.png` → `photo.jpg`）。
  この場合、元の `photo.png` は削除され、新しい `photo.jpg` が生成される
  （`photo.png.bak` は残る）。
- **出力は 8-bit**（全フォーマット共通で `-depth 8` に正規化）。webp は構造上
  8-bit のため、16-bit 入力をそのまま渡すと「可逆」が 16→8bit 変換で崩れる。
  これを防ぐため明示的に 8-bit 化している。よって UC2 の「可逆」は
  **8-bit での可逆**を意味する（webp は `cwebp -lossless -exact`、png は `oxipng`）。
- **JPEG 品質**は既定 `85`（`magick → cjpeg -quality 85 -optimize -progressive`）。
  環境変数 `OPT_IMAGE_JPEG_QUALITY` で変更可。実測の目安（写真・1800px）:

  | quality | 対元サイズ | ssimulacra2 |
  |---|---|---|
  | 80 | 約21% | 76.7 |
  | 85（既定） | 約26% | 79.6 |
  | 88 | 約31% | 81.9 |
  | 90 | 約47% | 85.5 |

  90 でサイズが跳ねるため、品質重視でも 85〜88 が実用的な上限。

## オプション一覧

| オプション | 意味 | 既定 | 備考 |
|---|---|---|---|
| `-f <jpg\|webp\|png>` | 出力フォーマット | `jpg` | jpg=非可逆 / webp・png=可逆 |
| `-c <色数>` | 減色の色数 | （減色なし） | UC2 のみ。`2/16/125/256/4096` |
| `-p <pixels>` | リサイズ画素（長辺） | `1800` | `-n` と排他 |
| `-n` | リサイズしない | — | `-p` より優先 |
| `-s <white\|dark>` | 署名（色指定） | （なし） | 右下に署名＋ライセンス |
| `-h` | ヘルプ | — | |

## 必要ツール

| ツール | 用途 | 導入 |
|---|---|---|
| `magick` (IM 7) | リサイズ・署名・減色 | 導入済み |
| `cjpeg`（mozjpeg） | JPEG 出力（UC1） | `brew install mozjpeg`（keg-only: `/opt/homebrew/opt/mozjpeg/bin/cjpeg`） |
| `cwebp` | WebP 出力（UC2） | 導入済み |
| `oxipng` | PNG 出力（UC2） | `brew install oxipng` |
| `pngquant` | ≤256 色の高品質減色（任意） | 導入済み |
